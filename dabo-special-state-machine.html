达播专场产品状态机

本页按当前达播专场排期系统原型梳理：专场主流程、模块派工、子任务、商务提醒、月度统计五套状态机。左侧为 Mermaid 源码，右侧为渲染流程图。

上传文件仅保存 点击提交后完成 延期和取消原因留存 月度统计由状态派生

完整文字版文档：GitHub Markdown 文档

4. 商务提醒状态机

红点提醒由子任务变化派生，提醒商务查看新增、上传和完成。

红点提醒 
stateDiagram-v2
  state noNotice as "无提醒"
  state newNotice as "新增提醒"
  state uploadedNotice as "附件已上传提醒"
  state doneNotice as "完成提醒"
  state cleared as "已读/处理"

  [*] --> noNotice
  noNotice --> newNotice : 新增子任务
  noNotice --> uploadedNotice : 上传附件
  newNotice --> uploadedNotice : 上传附件
  uploadedNotice --> doneNotice : 点击提交完成
  newNotice --> doneNotice : 直接提交完成
  doneNotice --> cleared : 商务查看或处理
  uploadedNotice --> cleared : 商务查看
  cleared --> noNotice

5. 月度统计派生流程图

月度统计不单独维护主状态，而是按取消、延期、完成、未完成的优先级派生。

数据中心 
flowchart TD
  start["月度统计读取专场记录"] --> canceledCheck{"专场是否已取消"}
  canceledCheck -->|是| canceled["已取消"]
  canceledCheck -->|否| delayedCheck{"专场或子任务是否延期"}
  delayedCheck -->|是| delayed["已延期"]
  delayedCheck -->|否| doneCheck{"是否已回填实际销售额或进入待复盘"}
  doneCheck -->|是| done["已完成"]
  doneCheck -->|否| unfinished["未完成"]
flowchart TD start["月度统计读取专场记录"] --> canceledCheck{"专场是否已取消"} canceledCheck -->|是| canceled["已取消"] canceledCheck -->|否| delayedCheck{"专场或子任务是否延期"} delayedCheck -->|是| delayed["已延期"] delayedCheck -->|否| doneCheck{"是否已回填实际销售额或进入待复盘"} doneCheck -->|是| done["已完成"] doneCheck -->|否| unfinished["未完成"]
已取消优先取消后不再计入完成率分母，取消原因必须留存。
延期第二优先专场延期或存在延期子任务时，统计为已延期。
完成第三优先回填实际销售额、进入待复盘或已有达标率时，统计为已完成。
